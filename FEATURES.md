# 功能清单

本文件记录 `fanqie-uploader` **当前代码层面已实现**的功能。

> ⚠️ 标注「已实现」指代码已写好；其中依赖番茄真实 DOM 的部分（定时发布、同步去重、
> 弹窗处理）**尚未在真实页面验证**，番茄改版可能导致选择器失效。真正确认「可用」需
> load 进 Chrome 实测。

---

## 一、核心上传流程 ✅

完整跑通「本地文件 → 番茄后台批量发布」：

1. **选文件夹** —— popup 里选本地章节文件夹（`webkitdirectory`，比 `showDirectoryPicker` 在弹窗里更稳）
2. **解析章节** —— 自动读取所有 `.txt` / `.md`，提取：
   - 章节号（文件名 `第N章` → 正文首部兜底）
   - 标题（首个非空行 ≤40 字优先，否则文件名）
   - 正文内容（标题行之后；自动清理 Markdown 标记）、字数
   - 按章节号**自动排序**保证发布顺序
3. **勾选** —— 任务列表带复选框、全选、已选计数
4. **串行发布** —— 一章一个标签页：开 → 填表 → 发布 → 关 → 下一章（避免并发被风控）

## 二、三种发布模式 ✅

popup 可选，由 `uploader.computePublishTime()` 计算时间、`publisher.handlePublishDialog()` 落地：

| 模式 | 行为 |
|---|---|
| ⚡ 立即发布 | 审核通过即发 |
| 🤖 智能定时 | 每天 N 章（1–10），从指定时刻起按小时递增 |
| ⏰ 自定义 | 指定首章时间，后续每小时一章 |

智能定时示例：每天 3 章、10:00 起 → 第1章今天10:00、第2章11:00、第3章12:00、第4章明天10:00……

## 三、健壮性功能 ✅

- **已上传同步去重** —— 进章节管理页时抓取表格（`.arco-table-tr`），按章节号/标题匹配，已发布的自动跳过，不重复发
- **失败自动重试** —— 失败章节自动重发，默认最多 3 次（popup 可关）
- **自动处理弹窗** —— 发布时自动点掉错别字提示、二次确认等对话框
- **绕过 React 受控组件** —— 原生 setter + 逐字符派发 `input` 事件填表；正文按段落塞进 ProseMirror 富文本
- **SPA 路由监听** —— `MutationObserver` 监听番茄单页应用页面跳转，自动接管

## 四、工程基础 ✅

- **MV3 标准结构** —— manifest + service worker + 2 个 content script + popup，纯手写无框架，加载即用
- **消息编排** —— background 做消息总线，负责开/关标签页、转发完成信号、桌面通知
- **状态浮标** —— 番茄页面右上角实时显示上传进度（彩色分级）
- **扩展图标** —— 16/32/48/128 四尺寸（番茄红 + 上传箭头）
- **桌面通知** —— 全部完成后弹通知

## 文件对照

| 文件 | 职责 |
|---|---|
| `manifest.json` | MV3 配置、权限、图标 |
| `background.js` | 消息总线 + 标签页编排 + 通知 |
| `content/uploader.js` | 调度大脑：同步去重 + 时间计算 + 串行 + 重试 |
| `content/publisher.js` | 发布手：填表 + 定时 + 发布 + 处理弹窗 |
| `popup/popup.html` · `popup.js` | UI：选文件夹 + 设置 + 勾选 + 启动 |

## 工程优化（已完成）

- [x] **选择器集中配置**：番茄 DOM 选择器统一放在 `publisher.js` 顶部 `SEL` 对象，改版只改一处
- [x] **`waitFor` 等条件**：关键处用轮询等元素出现/消失替代写死的 `delay`，更稳更快
- [x] **防重复发布**：失败重试前先回章节管理页核对该章是否已存在，已存在则跳过（不重发）
- [x] **侧边进度面板（sidePanel）**：实时显示每章状态、完成/失败/待发统计、进度条，含「停止」按钮
- [x] **多更时间错开**：每日≥2章时用时间选择器设置不同时刻（best-effort，依赖番茄 DOM）
