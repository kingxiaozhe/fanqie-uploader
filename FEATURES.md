# 功能清单

`fanqie-uploader` 当前已实现的功能总览。工具靠**模拟网页操作**驱动番茄作者后台，
依赖番茄真实 DOM；番茄改版后选择器需对照调整（已集中在 `publisher.js` 顶部 `SEL`）。

---

## 一、核心上传流程

1. **选文件夹**：popup 选本地文件夹，读取所有 `.txt` / `.md`
2. **解析章节**：章节号（文件名 `第N章` → 正文兜底，兼容空格）、标题（首个非空行）、
   正文（标题行之后，自动清理 Markdown 标记）、字数；按章节号排序
3. **勾选 / 预览 / 编辑**：列表可全选、可点 ✏️ 改标题/正文、可展开「排期预览」看每章发布时间
4. **串行发布**：一章一个标签页，开 → 填表 → 处理弹窗 → 发布 → 关 → 下一章

## 二、发布模式

- **⚡ 立即发布**：审核通过即发
- **🤖 智能定时**：每天 N 章，当天内从起始时刻按 `24/N` 小时均分；
  起始日期支持「自动接续已排期 / 指定 / 明天」
- **⏰ 自定义**：指定首章时间，后续每小时一章

定时由 `computePublishTime()` 计算，`publisher.js` **模拟点击日历/时间面板**落地。

## 三、反识别（降低被判定为工具）

- **操作节奏**：正常 / 慢一点 / 很慢（统一放慢所有延迟）
- **拟人随机延迟**：逐字输入 30~120ms 随机 + 偶发停顿 + 所有延迟 ±40% 抖动
- **章节间隔随机**：每章之间在用户设定的 `[最短,最长]` 秒区间内随机
- **发布时间随机偏移**：定时时刻 ±N 分钟随机，破整点规律
- **发布量上限**：本次最多发 X 章，到量自动停（风控更看频率）

## 四、稳定性与容错

- **同步去重**：进章节管理页优先调「章节列表」接口拉全量·跨分页已创建章节，已发章节自动跳过（接口不可用回退 DOM 抓当前页）
- **防重复**：点过"下一步"（章节已创建）后失败不重试；空正文在建章前中止
- **看门狗**：单章 5 分钟无响应自动按失败处理，绝不永久卡死
- **风险检测等待**：番茄"正在检测风险内容"阶段冻结超时、耐心等
- **多信号判成功**（跳转/离开发布页/弹窗消失/成功提示）+ 超时兜底
- **失败自愈**：弹窗未关时自动重点"确认发布"
- **停止**：浮标 / popup / 进度面板随时停，提交前响应零副作用
- **断点续传**：重进页面提示继续未发章节
- **失败自动重试**（默认 3 次，可关）

## 五、面板与工具

- **进度面板（sidePanel）**：每章状态、完成/失败/待发统计、进度条、停止、🔁 重发失败章、📄 导出 CSV 报告
- **桌面通知**：章节失败 / 到量自动停 / 全部完成
- **设置持久化**：所有发布设置记住上次选择（试填模式除外）
- **暗色模式**：popup / sidePanel 跟随系统深色
- **扩展图标**：`icons/`，由 `icons/icon.svg` 生成

## 文件对照

| 文件 | 职责 |
|---|---|
| `manifest.json` | MV3 配置、权限、图标、sidePanel |
| `background.js` | 消息总线 + 标签页编排 + 通知 |
| `content/uploader.js` | 调度：同步去重 / 排期 / 串行 / 看门狗 / 防重复 / 停止 / 续传 / 浮标 |
| `content/publisher.js` | 执行：填表 / 处理各弹窗 / 日历时间选择 / 确认发布（选择器集中在 `SEL`） |
| `popup/` | 选文件夹 / 设置 / 勾选 / 预览 / 编辑 / 排期预览 |
| `sidepanel/` | 实时进度 / 停止 / 重发 / 导出报告 |
