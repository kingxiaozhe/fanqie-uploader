# LESSONS（缺陷修复沉淀）

- [已结构化] **隐藏标签页的定时器会被 Chrome 节流到每分钟 1 次（隐藏>5 分钟触发 intensive
  throttling）**：所有"tick 计数 × 间隔"式超时在 `active:false` 打开的页面里都会被拉长几十倍。
  超时一律用墙钟（`Date.now()` 差值），需要"冻结等待"就累计冻结毫秒数扣除。
  ——来源：20260715-zombie-publish-tab

- [已结构化] **"读页面表格拿数据"的逻辑只对当前分页有效**：本项目已第二次栽在这里
  （第一次是同步去重，本次是排期接续）。凡需要"全量视角"的读数（最晚日期、是否已存在、
  总数），一律优先走章节列表接口（跨分页），DOM 只作回退并在日志标注降级。
  ——来源：20260715-daily-limit-schedule-collision

- [仅记忆] 平台返回的业务错误码要分类消费：-1010（限流，暂时性）已有自适应降速，
  -1020（每日上限，当日终局）需要改变排期策略而不是重试。新错误码出现时先问
  "重试有没有意义、影响的是这一章还是这一天"。
